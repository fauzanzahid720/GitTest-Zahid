Berikut adalah daftar istilah Git yang muncul dalam lecture ini, beserta penjelasannya dengan bahasa yang **sangat mudah dimengerti**:  

---

### **A**
#### **Add (git add)**
ğŸ“Œ **Definisi:** Perintah Git untuk menambahkan file ke **staging area**, sebelum akhirnya di-commit.  
ğŸ“Œ **Analogi:** Seperti menaruh barang di dalam keranjang belanja sebelum menuju kasir.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git add file.txt  # Menambahkan satu file
git add .         # Menambahkan semua file di folder saat ini
```

---

### **B**
#### **Branch (Cabang)**
ğŸ“Œ **Definisi:** Cabang dalam Git yang memungkinkan pengembangan proyek secara terpisah tanpa mengganggu branch utama.  
ğŸ“Œ **Analogi:** Seperti menulis draft dokumen baru tanpa mengubah dokumen asli.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git branch fitur-baru  # Membuat branch baru bernama "fitur-baru"
git switch fitur-baru  # Berpindah ke branch "fitur-baru"
```

---

### **C**
#### **Checkout (git checkout)**
ğŸ“Œ **Definisi:** Perintah lama untuk berpindah branch atau commit tertentu dalam Git.  
ğŸ“Œ **Analogi:** Seperti berpindah ke halaman lain dalam buku catatan proyek.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git checkout fitur-baru  # Berpindah ke branch "fitur-baru"
```
ğŸ“Œ **Catatan:** Sekarang lebih disarankan menggunakan `git switch`.

#### **Commit**
ğŸ“Œ **Definisi:** Menyimpan perubahan yang telah ditambahkan ke dalam **staging area** ke dalam repository.  
ğŸ“Œ **Analogi:** Seperti menekan tombol "Save" pada dokumen.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git commit -m "Menambahkan fitur login"
```

#### **Config (git config)**
ğŸ“Œ **Definisi:** Perintah untuk mengatur konfigurasi Git, seperti username dan email.  
ğŸ“Œ **Analogi:** Seperti mengatur profil sebelum mulai bekerja di sebuah proyek.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git config --global user.name "John Doe"
git config --global user.email "johndoe@example.com"
```

---

### **H**
#### **HEAD**
ğŸ“Œ **Definisi:** Pointer yang menunjukkan commit atau branch yang sedang aktif saat ini.  
ğŸ“Œ **Analogi:** Seperti tanda buku yang menunjukkan halaman terakhir yang sedang dibaca.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git reset --hard HEAD~1  # Kembali ke commit sebelumnya
```

---

### **I**
#### **Init (git init)**
ğŸ“Œ **Definisi:** Perintah untuk menginisialisasi repository Git di dalam folder.  
ğŸ“Œ **Analogi:** Seperti membuat folder baru khusus untuk menyimpan catatan perubahan proyek.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git init
```

---

### **L**
#### **Log (git log)**
ğŸ“Œ **Definisi:** Menampilkan riwayat commit dalam repository.  
ğŸ“Œ **Analogi:** Seperti melihat buku catatan yang berisi semua perubahan dalam proyek.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git log  # Menampilkan daftar commit
git log -p  # Menampilkan detail perubahan di tiap commit
```

---

### **M**
#### **Merge (git merge)**
ğŸ“Œ **Definisi:** Menggabungkan perubahan dari satu branch ke branch lain.  
ğŸ“Œ **Analogi:** Seperti menggabungkan dua versi dokumen menjadi satu.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git merge fitur-baru  # Menggabungkan branch "fitur-baru" ke branch saat ini
```

#### **Master (Branch Master)**
ğŸ“Œ **Definisi:** Branch utama dalam repository Git.  
ğŸ“Œ **Analogi:** Seperti dokumen utama yang menjadi acuan dalam proyek.  
ğŸ“Œ **Catatan:** Sekarang branch utama sering dinamakan `main`, bukan `master`.

---

### **R**
#### **Remote**
ğŸ“Œ **Definisi:** Repository Git yang disimpan di server online seperti **GitHub, GitLab, atau Bitbucket**.  
ğŸ“Œ **Analogi:** Seperti Google Drive untuk menyimpan proyek agar bisa diakses orang lain.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git remote add origin https://github.com/user/repo.git
```

#### **Repository**
ğŸ“Œ **Definisi:** Tempat penyimpanan semua file dan riwayat perubahan proyek yang dikelola dengan Git.  
ğŸ“Œ **Analogi:** Seperti folder proyek yang menyimpan semua file dan catatan perubahannya.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git init  # Membuat repository baru
```

#### **Reset (git reset)**
ğŸ“Œ **Definisi:** Menghapus commit terbaru dan kembali ke kondisi sebelumnya.  
ğŸ“Œ **Analogi:** Seperti menekan tombol "Undo" dalam editor teks.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git reset --hard HEAD~1  # Menghapus commit terbaru
```

---

### **S**
#### **Staging Area**
ğŸ“Œ **Definisi:** Tempat penyimpanan sementara sebelum commit dilakukan.  
ğŸ“Œ **Analogi:** Seperti tempat persiapan sebelum memasukkan file ke dalam repository.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git add file.txt  # Menambahkan file ke staging area
```

#### **Status (git status)**
ğŸ“Œ **Definisi:** Menampilkan informasi tentang status perubahan dalam repository.  
ğŸ“Œ **Analogi:** Seperti mengecek daftar tugas yang belum selesai.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git status  # Menampilkan status file yang sudah atau belum di-track
```

#### **Switch (git switch)**
ğŸ“Œ **Definisi:** Perintah baru untuk berpindah branch, menggantikan `git checkout`.  
ğŸ“Œ **Analogi:** Seperti berpindah ke jalur kerja yang berbeda tanpa menghapus pekerjaan yang sedang berlangsung.  
ğŸ“Œ **Contoh Perintah:**  
```bash
git switch fitur-baru  # Berpindah ke branch "fitur-baru"
```

---

### **T**
#### **Tracked & Untracked Files**
ğŸ“Œ **Tracked Files:** File yang sudah dikenal Git dan sedang dilacak perubahannya.  
ğŸ“Œ **Untracked Files:** File yang belum pernah ditambahkan ke dalam Git.  
ğŸ“Œ **Analogi:** Seperti catatan yang sudah masuk dalam arsip (tracked) dan catatan yang belum dicatat (untracked).  
ğŸ“Œ **Contoh Perintah:**  
```bash
git status  # Menampilkan file yang belum di-track
```
